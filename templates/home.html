Welcome to Smart Inventory Management (SIM)

SIM is a web application designed primarily for Humanitarian Organizations to increase efficiency and accuracy of Asset and Stock Management. Current version only includes asset management module and stock management is under development.

The application is open source and free to use. The source files can be found here. If you have any questions regarding the code feel free to contact via github with the developer.

Framework and APIs

SIM uses powerful Django Python Framework as the main build component. Datatables jQuery plug-in is used for table views and Bootstrap 4 and Semantic UI are used as main design elements. In short a variety of programing languages are used to make this happpen.

Postgres Database are used in this production sample. Application is running on a free Heroku server as it can be told from the URL

Additional information in detail can be found under the relavent section for devolopers.

How does it work?

If you are already working in humanitarian logistics you will find the terminology used in the application quite familiar. Like all organizations have their differences in their process, SIM is based on a fusion of "best practices" in inventory management and might be a bit different than what you are already using.

There are 6 main sections for Asset management. Brief explanation of each can be found below:

Assets:

As you can imagine from the name, it is the main function where you can view, create, assign and update assets recorded in your database

Users:

Here you can view, create, assign and update users(employees) of your organization. This "user" class is seperate from users of the application and is reffered as "custodian" in the code to avoid overlaps with django user class.

Reports:

Under this section you can select a user or an asset to generate a report. In user reports, you will find the all the assets assigned to this user in a chronological order with dates. In asset reports you will find in chronological order users of this asset and the date of assignment

Important!!: Report entries are automatically generated when and asset issuance is made. These records can only be accessed modified and deleted through admin panel which will be explained in detail later.

Allocations:

This is where end of grant allocations are done. Once an allocation is created, Donor and Budget line of the asset will be automatically changed based on the allocation information entered on "create".

Allocations can be viewed in list and in detail where list of allocated assets can be found in a table including the prior donor information. Also allocation information can be found in asset details as well

Disposal:

Under this section, you can select assets to dispose from the master database referring to a document number and mode of disposal. At the moment disposed assets still remain in the master database. Based on ypur organizations policy python code can be modified:

- To remove the asset and save it to another database table called "disposed".

- To change the assets condition to disposed

Categories:

Here you can view and create categories and subcategories. Update and delete functions can only be done through admin panel(info below)

Django Admin

Django Admin panel is a User interface to manage database tables and models created under this application. As default it is not "user friendly" but is a valuable tool to fix major issues that can occure in the database. Some functionalities are only accessible via admin database such as delete and update of categories and subcategories.

Through admin panel, it is possible to import data from various file formats(excel, csv) as well as export

Table Functionalities

This application uses "Datatable" j-query Plug-In to visualize database tables in the most customizable way possible.

Viewing Data:

By default, tables are set to display 50 items per page. It can be reduced or increased based on your needs.

Search Function:

- You can search the all rows and colums from the search bar on the upper right corner.

- Each column has a searchbar where you can use it as "filter".

- Searches are done in realtime no need to hit any key.

- Searches will apply for the whole table, not only for visible items.

Column Customization:

- Column width can be resized based on your need.

- Columns order can be changed via drag the column title.

- In assets main view, columns can be hidden through a popup window that comes up onclick to Hide Column button.

-"Column hide preferences" would be saved for 1 hour. Preferences will stay even if the page is refreshed.

- Above preferences will be reset onclick to "Reset Table" button.

Excel Export:

- You can export the view you filtered as an excel file.

- Hidden columns will not be reflected on the excel file.
